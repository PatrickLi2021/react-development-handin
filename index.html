Assignment Overview

The main purpose of this assignment are to practice conducting competitive analyses between competitor companies in order to learn how to idenfiy strengths and weaknesses in design eleemnts used by industry designers. Additionally, in order to put these skills into effect, I created a function aggregator interface using React to practice using a reactive framework for frontend development.

Competitive Analysis Summary

To begin, I think that EdStem's favoriting platform was the least well-designed in terms of number of features and usability. In total, EdStem just gave users the ability to "star" posts that they deemed important or to look at later. However, there was no way to organize these starred posts or filter them in any way (i.e. by keyword). Spotify's web player and YouTube were very similar in terms of the features and usability of their favoriting features. All of them had the capability to view the number of items in the list, remove items from the list easily, and sort by date added (Spotify was able to do this on the app, not the website however). In the end, what made YouTube slightly better designed was that it allowed the user to immediately view the list after the item was added while in Spotify, you had to manually navigate to the Liked Songs playlist in order to do this.

List of Features to be Implemented in React Website

After performing the competitive analysis, I definitely want to implement the ability to sort and filter items in my list, as I believe that that will make it easier for the user to locate items within the list, especially if there are a lots of items. I also want to create separate pages for the available items and the items to be added and have a feature that allows the user to quickly and easily view the items in the list/cart once they add one. The last feature that I believe is core to a well-designed and usable aggregator website is the ability for the user to easily remove items from their list and receiving confirmation that they have done so successfully.

Part 2: Functionality and List/Aggregator Interface

Overview

For this part of the assignment, I created a list-based interface/website using HTML, CSS, and React/JavaScript. The theme that my website revolves around is the European soccer transfer market, which incorporates both lists (to be used for filtering, sorting, and resetting) and an aggregator (adding players to a team that you create).

Intention of the App

The intention of the app is to simulate the actions of a soccer managaer in the transfer window. The transfer window is a period of time during the soccer season in which managers and clubs have the ability to buy players to add to their team. Each player in the market has an associated transfer value that represents how expensive they would be to add to your team.

How this Website Uses Filtering, Sorting, and Aggregation

  • Filtering:

    • This website uses filtering because each player has a position (i.e. striker, left winger, etc.) and the user can filter the available players by their position. For example, if I am the manager of a club and I am specifically looking to add strikers to my team, I can filter the available players to only display strikers.

  • Sorting:

    • Sorting is incorporated into the website because managers are able to sort the market by price, allowing them to view the least expensive to most expensive players and vice versa.

  • Aggregating:

    • Lastly, managers can aggregate players together into their team, which is shown at the bottom of the page. Each player that they purchase is added to their team and they can see those players along with the total amount of spending needed to purchase all of them.
